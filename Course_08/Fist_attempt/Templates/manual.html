Textmining

The first part of this process involves retrieving relevant keywords for co-occurrence. This is done by opening a text file and extracting the keywords needed for further analysis.

After getting the keywords, we move on to the next stage, where we open a database and extract the abstracts. These abstracts likely contain important information that we want to analyze.

To further refine the data, in step 3 we will work with a for loop to remove stem words and stop words per abstract. This eliminates words that add little value to the analysis, such as common words or words with no specific meaning. We also remove special characters to keep the data clean and structured.

Then it is time for step 4, where we will find the relevant keywords in the abstracts. This can be done, for example, by using advanced search algorithms or by comparing the words in the abstracts with the retrieved keywords.

Now that we have identified the necessary keywords, we can move on to step 5, where we perform co-occurrence analysis. We construct a co-occurrence matrix using the keywords found. This matrix indicates how often the keywords appear together in the abstracts and helps identify patterns and relationships between the words.

In step 6, we add the co-occurrence matrix to a larger master_matrix. This allows us to combine the results of different analyzes and get a more comprehensive picture of the relationships between the keywords.

To store the results, let's move on to step 7, placing the master_array in a JSON file. Using JSON (JavaScript Object Notation) makes it easy to store and share structured data.

Finally, in step 8, we read out the JSON file and store the interesting results in a table. This allows us to present key findings and make them easily accessible for further analysis or reporting.

This process provides a structured method to extract keywords, analyze co-occurrence, and store key findings for further processing and interpretation. It can be useful in exploring large amounts of textual data and discovering hidden connections between words and concepts.

Database

The database functionality of the Bioinformatics Tool allows you to store article information, such as titles, abstracts, authors, and publication dates. You can also link articles to keywords for better organization and retrieval.

The database headers with the description contained in them.

Headers Description
Articles Unique identifier for each article.
article_ID Unique identifier for each article.
title Title of the article.
abstract Abstract of the article.
links Link to the article on PubMed.
pubmed_id PubMed ID of the article.
authors Authors of the article.
dates Publication date of the article.
Articles_Keywords Maps articles to keywords using a many-to-many relationship.
Articles_article_ID Foreign key referencing the article_ID column in the Articles table.
Keywords_keyword_ID Foreign key referencing the keyword_ID column in the Keywords table.
Keywords Stores unique keywords.
keyword_ID Unique identifier for each keyword.
keyword Keyword associated with articles.
